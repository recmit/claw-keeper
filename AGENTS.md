# Agent Working Notes

These notes are for AI agents and other automated assistants working in this
repository.

## Required Checks

- Run `scripts/check` before committing.
- Install the tracked Git hooks once per checkout with `scripts/install-git-hooks`.
- Do not bypass the pre-commit hook with `--no-verify` unless the user explicitly
  asks for it.

## Versioning

- Bump the package version for every repository change that is meant to be
  committed.
- Update both `pyproject.toml` and the matching `claw-keeper` package entry in
  `uv.lock`.
- The pre-commit hook checks that the staged version differs from `HEAD` and
  that `pyproject.toml` and `uv.lock` agree.
- Use patch bumps for routine fixes, documentation, tests, and internal
  maintainability changes. Use minor bumps for new user-facing behavior, and
  major bumps only for intentional breaking changes.
- Keep the version bump in the same commit as the change it describes.

## Project Boundaries

- Keep the public tool repo separate from private OpenClaw state history.
- Never add credentials, SSH private keys, OAuth material, raw logs, session
  data, completion databases, or other OpenClaw runtime secrets to this repo.
- Treat `skills/claw-keeper/SKILL.md` as the canonical OpenClaw skill source.
  The `openclaw-skill` branch is generated by GitHub Actions.

## Local Workflow

- Prefer small, focused changes that match the existing CLI and test style.
- Add or update tests when changing behavior.
- Use `rg` for code searches and `scripts/check` for linting and tests.
